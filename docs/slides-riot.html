利用した事例

10ヶ月ぐらいまえに作りかけた資料

.

最大の特徴 : カスタムタグ

  • .tagファイル
  • HTMLのタグを定義する感じのファイル
  • tagファイルの中にjs / cssまで書く
require('./sector')
<app>
  <sector title='とりあえず概要'>
    この資料に関する説明です
  </sector>
</app>

<sector>
  <h2>{ title }</h2>
  <p>{ message }</p>
  <small>
    <yield />
  </small>
  this.title = opts.title
  <style media="screen">
    sector {
      display: block;
      padding: 50px 0 30px;
    }

    h2 {
      font-size: 50px;
      margin-bottom: 0;
    }
  </style>
  <script>
    this.message = 'Hello Riot'
  </script>
</sector>
.

他の機能

  • eachもちゃんとある
  • routerもちゃんとある
  • obserbableもriotの中にある
.

好きなところ

  • html / css / jsがそのままダラダラ書ける
  • 日本語のドキュメントがある
  • css_modulesチックなことを考えなくても、タグごとに名前空間が別れる
.

ちょっとなあってところ

  • どうしても大規模な設計に向かない
  • タグの継承の概念がない(サブクラス的なタグが作りたい)
  • obserbableの書き方がいつもとちがうし違和感
.

普通のobserbable

いつもと違うなあ…

var foo = riot.observable();
foo.on('myEvent', function(data) {
  // your code here
});
foo.trigger('myEvent', { param: 1 });
